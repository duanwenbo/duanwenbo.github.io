就算大雨让这座城市颠倒，也要保持微笑🤗

第二次，淘气的代码在傍晚8点，消失的无影无踪。 掩卷沉思，吟小诗一首。

问题复现：

  • 设置

    • VS code 1.78.2 (Universal)

    • 开启auto save, afterDelay 默认 1000 ms

  • 复现

    • 将外层父文件夹同名jupyter文件复制进入子文件夹，开始修改。

    • 未手动保存, cmd+W 关闭窗口。

    • TIMELINE 功能在开启autosave的情况下未启用，提示 No timeline information was provided。

解决方法：

  • 问题只发生在当被复制对象包含大量输出结果时
  • 清空所有结果输出，自动保存恢复

总结：

  • 养成win/cmd+S的习惯

  • 避免混淆项目文件

  • VS Code 临时备份地址 ~/Library/Application\ Support/Code/Backups

  • VS Code 全局搜索 Command + Shift + F

  • VS Code 本地设置 Command + Shift + P
